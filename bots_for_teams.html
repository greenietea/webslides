Share the story about how and why I built bots.

Two years ago...

I started analysing conversations archived in HipChat and Slack.

How I started

  • All the words

    Downloaded the archived conversations in text format and clean up the data to remove private channels and notifications from Slack integrations

  • NLTK

    Wrote a script in Python and used the Natural Language ToolKit (https://nltk.org) to parse the files to see what it finds

  • Uncovered frequently used term

    Chose a list of excluded words and find that most frequently asked questions are about clients

  • Simplified

    Instead of using NLTK, a simple hash table worked just as well

Connecting the dots


Why don't they just use the platform I built? It's all got the data!

Interface with a bot

  • Intent

    What type of info is the user asking for?

  • Parameter

    What is specific about the intent? (e.g. client name)

  • Action

    How to map the intent to an action for the bot?

  • Request data

    Which data source does the bot request data from?

  • Response

    Parse, prettifier, return response to the user

Results


Engaged users

Used in group discussions

Users asking more questions

Challenges


Imperfect users with strict strings matching

Lack of parameters

Context switching

How I really did it at the end


Used a service to do the request parsing Api.ai https://api.ai

Lessons learned


Software engineers are problem solvers first and software is one of the tools.

Don't reinvent the wheel.

Geek out on something that engages the users first, and leave the hard computing problems for later.
